Create a modern, clean, and user-friendly Employee Attendance Web App UI for “Bianore Attendance System”. 
This web app is for employees and will be connected to an Admin Attendance Dashboard.

Design style:
- Modern SaaS dashboard design
- Clean layout, easy to understand, not too crowded
- Professional HR / attendance system look
- Use soft neutral background with strong primary color accent
- Rounded cards, clean icons, clear typography
- Prioritize usability and fast employee actions
- Responsive web app layout
- Make the UI suitable for desktop web, but still easy to adapt to tablet/mobile
- Use clear status colors:
  - Green for on-time / present
  - Yellow or orange for late
  - Red for absent / warning
  - Blue for information / active state

Main pages / screens to design:

1. Login Page

Create a secure and modern login page for employees.

Content:
- Company logo: Bianore Attendance
- Welcome text
- Login form with:
  - Email or phone number input
  - Password input
  - Login button
- Alternative security options:
  - Login with Google Authenticator
  - OTP / 2FA verification
- “Remember device” option
- Forgot password link
- Short security note: “Your device and login activity are protected”
- Clean illustration or abstract visual related to work attendance / employee productivity

Also create additional authentication flow screens:
- OTP verification screen
- Google Authenticator code screen
- Device binding confirmation screen
- Role selection or role indicator after login:
  - Admin
  - HR
  - Employee

2. Employee Home Dashboard

Purpose:
The main page for employees to check attendance quickly and see today’s work status.

Main features:
- Check-in button
- Check-out button
- Today status card:
  - On-time
  - Late
  - Absent
- Today work time / working duration
- Current time and date
- Quick summary:
  - Check-in time
  - Check-out time
  - Total work duration
  - Overtime indicator if any
- Small profile card:
  - Employee name
  - Department
  - Role
  - Attendance status today

UX requirement:
Make check-in and check-out the most visible actions on the page.
Use a large primary button and clear status indicator.

3. Work Schedule Page

Purpose:
A personal attendance calendar for employees.

Main features:
- Monthly attendance calendar
- Highlight status by date:
  - Present
  - Late
  - Absent
  - Leave
- Summary cards:
  - Total present days
  - Total late days
  - Total absent days
  - Total leave days
- Calendar legend with simple colors
- Attendance detail when clicking a date:
  - Date
  - Check-in time
  - Check-out time
  - Status
  - Work duration

UX requirement:
Make the calendar visually clear and easy to read.
Use simple color indicators, not too many details inside each calendar cell.

4. Task / Activity Page

Purpose:
Employees can view daily tasks and update task progress.

Main features:
- Daily task list
- Task card with:
  - Task title
  - Description
  - Due time
  - Priority
  - Status
- Status update options:
  - Not started
  - In progress
  - Completed
- Filter by status
- Add short progress note
- Activity summary:
  - Total tasks
  - Completed tasks
  - Pending tasks

UX requirement:
Design the task list like a simple productivity board.
Make task status easy to change with buttons or dropdown.

5. Sales Calling / Field Visit Page

Purpose:
For field employees or sales team to record customer visits with GPS validation and proof.

Main features:
- Start visit / check-in location button
- GPS location detection card
- Upload photo proof of visit
- Add visit notes
- Customer / client name input
- Visit purpose input
- Submit visit report button
- Visit history list

Visit history card should include:
- Client name
- Visit date and time
- Location
- Photo proof thumbnail
- Visit note
- Status: Submitted / Verified / Flagged

UX requirement:
Make this page feel like a field report form.
Important actions should be simple and fast because employees may use it while outside the office.

6. Payroll / Payslip Page

Purpose:
Employees can view salary slips and payment history transparently.

Main features:
- Latest payslip card
- Download payslip button
- Payment history table
- Salary component details:
  - Basic salary
  - Allowance
  - Overtime
  - Deduction
  - Late deduction
  - Tax
  - Net salary
- Month selector
- Payslip status:
  - Paid
  - Pending
  - Processing

UX requirement:
Use a clean financial layout.
Make salary details easy to scan and separate earnings from deductions clearly.

7. Notification Page

Purpose:
Employees receive attendance reminders and important company updates.

Main features:
- Notification list
- Notification categories:
  - Attendance reminder
  - Late notification
  - Important update
  - Payroll update
  - Task update
- Notification card with:
  - Icon
  - Title
  - Short message
  - Time
  - Read / unread status
- Mark all as read button
- Filter by category

UX requirement:
Make unread notifications visually clear but not too aggressive.
Use clean cards and simple icons.

8. Employee Profile / Account Security Page

Purpose:
Employees can view account and security information.

Main features:
- Employee information:
  - Name
  - Employee ID
  - Department
  - Role
  - Email
  - Phone number
- Security settings:
  - Google Authenticator status
  - OTP / 2FA status
  - Device binding status
  - Active device list
- Logout button

UX requirement:
Make this page simple, secure, and professional.

Navigation structure:

Use a left sidebar navigation for desktop:
- Home
- Work Schedule
- Task / Activity
- Sales Calling / Field Visit
- Payroll / Payslip
- Notification
- Profile / Security

Sidebar behavior:
- Active menu should be clearly highlighted
- Use simple modern icons
- Keep menu labels readable
- Include employee profile mini card at the bottom

Top bar:
- Page title
- Search or quick action area if needed
- Notification icon
- Employee avatar

Dashboard visual direction:
- Use cards for every key information
- Use status badges for attendance states
- Use clean tables only when needed
- Avoid complicated charts unless useful
- Use whitespace generously
- Make the employee workflow very obvious:
  1. Open app
  2. Check attendance status
  3. Check-in or check-out
  4. View schedule / task / payroll / notification

Important design priority:
The most important goal is usability.
The employee should understand what to do within 3 seconds after opening the app.
The UI must feel modern, professional, and trustworthy for a company attendance system.

Generate complete UI screens with realistic sample data, modern components, and polished visual hierarchy.
